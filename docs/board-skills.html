Queues, and the skills on each transition

Columns are resting states — the name is true of every card in them, always. Agents run on the transitions: a drag is the launch, the card stays in its origin queue with a live badge, and it arrives in the next queue only when the stage completes. The one exception is deliberate: Code is a live activity lane — the meat of the pipeline gets its own column, and cards leave it on their own when the build lands. State on the column, verb on the drop target.

IDEAS ➜ PRODUCT BACKLOG
Drag opens the ideation chat (evolve mode, guided): an in-daemon agent challenges the idea, rewrites the same ticket, and removes the human/idea label. no marker — the label swap moves the card
PRODUCT ➜ ENGINEERING BACKLOG
Launches /human-plan <KEY>: planner + verify agents draft the engineering plan; it lands as a [human:plan] comment. While it runs the card stays here with ◐ planning…. [human:planning-started] → [human:plan-ready]
ENGINEERING ➜ CODE
Launches /human-execute <KEY>: the executor works the plan and runs the human-done self-check. The card moves into Code for the whole build — the live activity lane. Failures stay here, red. [human:implementation-started]
CODE ➜ READY FOR REVIEW — AUTOMATIC
No drag, no drop target: when the executor posts the handoff, the card releases itself into Ready for review. Cards can only earn their way in. [human:ready-for-review]
REVIEW ➜ READY TO DEPLOY
Launches /human-review <KEY>: the reviewer checks the branch against acceptance criteria and records the verdict. Card shows ◐ reviewing… in place. [human:review-started] → [human:review-complete]
OPEN PULL REQUEST
Terminal action is a button on the card — no agent; the daemon pushes the branch and opens the PR. [human:pr-started] → [human:pr-pushed]

◐ RUNNING = STAYS PUT
Planning and reviewing keep the card in its origin queue — a column never claims what isn't true yet. Code is the deliberate exception: the meat of the pipeline gets its own lane.
VERB ON THE TARGET
While dragging, the drop zone shows the action it launches (Define it · Plan it · Build it · Review it). State on the column, verb on the affordance.
