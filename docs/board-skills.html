IDEAS ➜ BACKLOG
Drag opens the ideation chat (evolve mode, guided) — an in-daemon agent that challenges the idea, then rewrites the same ticket and removes the human/idea label. no marker — the label swap moves the card
BACKLOG ➜ PLANNING
Launches /human-plan <KEY>: planner + verify agents draft the engineering plan; it lands as a [human:plan] comment (or a separate engineering ticket in split topology). [human:planning-started] → [human:plan-ready]
PLANNING ➜ IMPLEMENTATION
Launches /human-execute <KEY>: the executor agent works the plan step by step, runs the human-done self-check, and posts the review handoff. [human:implementation-started] → [human:ready-for-review]
IMPLEMENTATION ➜ VERIFICATION
Launches /human-review <KEY>: the reviewer agent checks the branch against the ticket's acceptance criteria and records the verdict. [human:review-started] → [human:review-complete]
VERIFICATION ➜ DONE
No agent — the daemon itself pushes the recorded branch and opens the pull request (PR body carries the ticket trail). [human:pr-started] → [human:pr-pushed]
