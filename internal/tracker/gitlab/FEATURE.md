# GitLab

Connects `human` to your GitLab issues so you can read and manage them from the command line. It picks up your GitLab instances from configuration, including tokens stored in a vault, and works across self-hosted and hosted GitLab.

- Lists issues, filtered by project and state
- Looks up a single issue by key
- Creates new issues with title and description
- Adds and reads comments on issues
- Opens or closes issues by transition
- Assigns issues to a user
- Edits an issue's title or description
- Connects to self-hosted or hosted GitLab
