YAML lens — the form and the file, side by side, always in sync

A full settings view split in two: a friendly form on the left, the actual .humanconfig.yaml on the right. Focusing a form field highlights the exact lines it writes; editing either side updates the other. The UI never hides that there is one config file — it teaches it, so what you click today you can commit, share, and diff tomorrow.

+ total transparency: no hidden second config store, git-diffable + teaches the YAML — the file stays shareable with the team/repo + power users can paste whole sections; form users never must − two-way sync + comment-preserving YAML writes are real work − raw YAML on screen can intimidate exactly the non-editor audience
