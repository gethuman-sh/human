Static mockups, no functionality. All five render the same config surface — trackers (shortcut human / linear work), the ticket-kinds mapping, vault, daemon, appearance — read from .humanconfig.yaml via daemon config-get and written back with config-set. They differ only in the container and navigation model. Amber sticky notes on each mockup explain the behavior.
